# Демонстрационный сервис с Kafka, PostgreSQL и кэшем

Информационная система для получения данных из очереди сообщений, их надежного хранения и быстрого предоставления через API.

## Видео демонстрация работы сервиса
[![Google Drive](https://img.shields.io/badge/Google%20Drive-Смотреть_видео-blue?style=for-the-badge&logo=googledrive&logoColor=white)](https://drive.google.com/file/d/1dhPQd-btcxyqww3a8az_FxTxUMqe0Soy/view?usp=sharing)

## Основные возможности
- Очереди сообщений: Прием данных из Kafka (Redpanda). Реализовано подтверждение только после успешной записи в базу, что исключает потерю сообщений.
- Хранение данных: Использование PostgreSQL с типом данных JSONB для сохранения сложных структур заказов.
- Ускорение доступа: Внутриигровой кэш в оперативной памяти. При запуске сервис автоматически «прогревает» кэш, загружая последние данные из базы.
- Безопасное завершение: При остановке (Ctrl+C) сервис корректно закрывает соединения с базой и брокером, дорабатывая текущие задачи.
- Интерфейс: Веб-страница для поиска заказов по идентификатору (ID).

## Технологический стек

Здесь используется следующий скрипт-эмулятор:

- Язык: Go (Golang)

- База данных: PostgreSQL

- Брокер сообщений: Kafka (Redpanda)

- Контейнеризация: Docker, Docker Compose

- Фронтенд: HTML, JavaScript (ванильный)

---
## Запуск

Система полностью настроена для работы в контейнерах. Для сборки и запуска всех компонентов выполните:

```docker-compose up --build```

---
## Доступные адреса:

Интерфейс пользователя: http://localhost:8081 (поиск заказов по ID)

Консоль управления Kafka: http://localhost:8080 (просмотр топиков и сообщений)

---
## Как это работает (алгоритм)

1. Эмулятор (продюсер): встроенный компонент генерирует случайные заказы и отправляет их в очередь.

2. Обработчик (консьюмер, подписчик): читает очередь, проверяет корректность данных и записывает их в БД.

3. БД и кэш: одновременно с записью в БД данные попадают в оперативную память.

4. API: при поиске заказа система сначала смотрит в кэш (мгновенно), и только если его там нет — обращается к базе данных.
